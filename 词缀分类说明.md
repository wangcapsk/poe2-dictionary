# 关于"词缀"分类的说明

## 📋 分析结果

在 PoE2DB 的数据中，**没有独立的"词缀"分类**。

## 🔍 相关数据分布

### 1. **Passive (天赋/被动技能)** - 1,172条
这是当前被归类为 `mods` 的主要来源。

**内容包括：**
- 天赋树上的被动技能节点
- 升华职业的特殊天赋
- 各种被动加成效果

**示例：**
```
一击中的 (Clean_Shot)
万事通 (Jack_of_all_Trades)
不念荣辱 (Honourless)
冰霜化身 (Embodiment_of_Frost)
```

### 2. **Keywords 中的词缀相关内容** - 7条

包含"词缀"关键词的条目都归类在 `Keywords` 下：

| 中文 | 英文 | 说明 |
|------|------|------|
| 减速强度词缀 | Slow_Magnitude_Modifier | 减速效果强度修正 |
| 分裂词缀 | Fractured_Modifiers | 分裂装备的特殊词缀 |
| 怪物词缀 | Monster_Modifiers | 怪物的强化词缀 |
| 最低词缀等级 | Minimum_Modifier_Level | 词缀最低等级限制 |
| 深渊词缀 | Abyssal_Modifiers | 深渊装备词缀 |
| 渎灵词缀 | Desecrated_Modifiers | 渎灵装备词缀 |
| 绑定词缀 | Bonded_Modifiers | 绑定的固定词缀 |

这些是**词缀系统的机制说明**，不是具体的词缀内容。

### 3. **增幅器 (Augment)** - 123条

这可能是最接近"装备词缀"的分类，但它指的是**塔防词缀（先驱秘藏机制）**。

**示例：**
```
乌拉曼的凝视 (Ulamans_Gaze)
健壮符文 (Robust_Rune)
冰川符文 (Glacial_Rune)
克林斯昂的回复符文 (Craiceanns_Rune_of_Recovery)
```

## 🎯 结论

### PoE2DB 数据中没有装备词缀数据库

**原因：**

1. **装备词缀数量巨大**
   - 前缀词缀数百种
   - 后缀词缀数百种
   - 不同装备类型有不同词缀池
   - 词缀有等级变种

2. **autocompletecb.json 的设计目的**
   - 这个JSON是为了**搜索自动补全**功能
   - 主要包含：技能名、装备名、怪物名、通货名
   - **不包含**：装备词缀、数值数据、详细描述

3. **词缀数据在其他地方**
   - 装备词缀数据通常在 poe2db.tw 网站的其他API
   - 可能在 items、mods、stats 等专门的数据文件中
   - 需要访问网站的其他数据接口

## 💡 当前网站的分类策略

### 我们的 6 大分类中，`mods` 包含：

- ✅ **Passive (天赋)**：天赋树被动技能 (1,172条)
- ❌ **装备词缀**：不在 autocompletecb 数据中

### 建议

如果需要装备词缀数据，有以下选项：

1. **保持现状**
   - 当前的 `mods` 分类实际上是"天赋/被动技能"
   - 这对玩家查询游戏内容已经足够

2. **重命名分类**
   - 将 `mods` 改名为 `passives` 或 `天赋`
   - 更准确反映实际内容

3. **寻找装备词缀数据**
   - 需要查找 poe2db.tw 的其他API端点
   - 可能需要爬取网站或使用其他数据源

## 📝 总结

✅ **autocompletecb.json 不包含装备词缀数据**  
✅ **当前 `mods` 分类主要是天赋被动技能**  
✅ **词缀相关的机制说明在 `Keywords` 中**  
✅ **网站的分类策略是正确的**
